📋 Étapes pour publier sur GitHub Pages

1 Créer le repo GitHub

Allez sur github.com/new ou utilisez la CLI :

gh repo create veille-ao-btp --public --source=. --remote=origin --push

Paramètres :

  • Repository name: veille-ao-btp
  • Owner: wecanmoove
  • Public (nécessaire pour GitHub Pages)

2 Configurer le remote et pousser

Si créé via l'interface web :

cd ~/OneDrive/data_engineer_associate/veille-ao-btp
git remote add origin https://github.com/wecanmoove/veille-ao-btp.git
git branch -M main
git push -u origin main

3 Activer GitHub Pages

Sur https://github.com/wecanmoove/veille-ao-btp/settings/pages :

  1. Section Pages (à gauche)
  2. Source : "Deploy from a branch"
  3. Branch : main
  4. Folder : /docs
  5. Cliquer Save
⚠️ Important : GitHub Pages déploiera automatiquement le contenu de /docs après quelques minutes.

4 Vérifier le déploiement

Après 2-3 minutes, votre landing page sera accessible à :

🎉 https://wecanmoove.github.io/veille-ao-btp/

🚀 Lancer l'application en local

# Installer les dépendances
npm install

# Configurer l'environnement
cp .env.example .env
# Éditer .env avec vos clés API (OpenAI, Resend, Slack)

# Initialiser la base de données
npm run db:push
npm run db:seed

# Lancer le serveur dev
npm run dev

# Ouvrir http://localhost:3000/ao dans le navigateur

📞 Support & Questions

En cas de problème :

  • Vérifier que le repo est public (Settings → Visibility)
  • GitHub Pages apparaît dans Settings seulement si le repo a du contenu
  • Le déploiement prend 2-3 minutes après activation
  • Vérifier le dossier /docs existe et contient index.html
